1. Introduction

The Bitsquid Documentation System is the system used by Bitsquid for all system documentation (source code documentation is done through Doxygen).

2. Syntax Overview

2.1 Structure

@title Document title Document title
# Level 1 heading Level 1 heading
## Level 2 heading Level 2 heading
### Level 3 heading Level 3 heading
* List item Item in (unnumbered) list
@note Text Text to display in note box
@img images/larch.png Include image
@caption The Larch Caption for image
:definition Term to define (definition is indented)
@th Row Row of table headers
@td Data Row of table data

Example of list:

  • One

  • Two

  • Three

Example of note:

Note

Don't forget!

Example of image:

A test image

Example of definition list:

Lua

A scripting language

C++

A low level language

2.2 Formatting

%i Italic word Italic word

%cc Multiple code words %cc Multiple code words
%link http://www.bitsquid.se/ http://www.bitsquid.se/
\%i Escaped %i Escaped

2.3 Source code

@lua ... @endlua Mark a number of lines as Lua code
@cpp ... @endcpp Mark a number of lines as C++ code
@code ... @endcode Mark a number of lines as generic code
@lualine print(x) Used for a single line of Lua
@cppline x++; Used for a single line of C++
@codeline cd ~/lib Used for a single line of generic code

Example of source code:

for (unsigned i=0; i<n; ++i)
	printf("%i\n", i);

2.4 Glossary

%g Term Glossary term
%gg Multiple word term %gg Glossary term with multiple worlds
%g(term) Text Glossary item where term differs from text
%gg(term) Multiple word term %gg Same for multiple words
@glossary ... @endglossary Encapsulates glossary definition

Example of glossary:

Don't forget to by groceries.

forgetting
The act of losing from one's memory that which has been held there.
groceries
Consumable goods purchased in a store.

2.5 API documentation

@api Enter API documentation mode
@text Back to regular text processing mode
%r Applicaton.quit() Function reference: Application.quit() 
@api

api_function_definition(with, arguments)
    Api function description

    :parameter
        With description

Example of api documentation:

api_function_definition(with, arguments)

Api function description

parameter

With description
